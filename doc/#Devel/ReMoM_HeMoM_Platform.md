### Footnotes

 * [1] ReMoM вҖ” the windowed game executable (the user-facing build of the ReMoM project). Linked against momlib + MOX + STU + Platform where Platform is whichever windowed backend the preset chose (SDL2 / SDL3 / Win32).

 * [2] HeMoM вҖ” Headless Master of Magic. A lean entry point hardwired to Platform_Headless, regardless of which windowed backend the preset selected. Supports --newgame, --load SAVE.GAM, --record game.RMR, --replay game.RMR.

 * [3] Platform layer вҖ” a single contract (Platform.h / Platform_Keys.h / Platform_Replay.h) with four implementations under /platform/{sdl2,sdl3,win32,headless}/. The engine never talks to SDL/Win32 directly; everything funnels through the contract.

 * [4] Headless mode вҖ” Platform_Headless provides no-op video/audio, a stub timer/EMM, and a programmatic input source. It's what makes deterministic CI runs and --continue save tests possible.

 * [5] Record & Replay вҖ” sits at the Platform_Event_Handler() boundary. On record, every engine-visible input frame is appended to a human-readable .RMR file (with a header line # random_seed=<seed> so the run is reproducible). On replay, the file is the input source вҖ” OS events are ignored. platform/replay/Replay.c:1-13

 * [6] Artificial Human Player (AHP) вҖ” src/Artificial_Human_Player.c. Reads a .hms scenario (wait, key, click, next_turn, quit, вҖҰ), and on each frame injects synthetic keys/mouse via the same platform entry points a real user's input takes. Linked into both ReMoM and HeMoM, so the exact same scenario can run windowed or headless. src/Artificial_Human_Player.h:1-19

 * [a] AHP is compiled into both targets; it stays dormant unless a scenario is loaded.


Program-flow paths

1. Regular ReMoM usage (a human plays the game)

User keyboard/mouse  вҶ’  SDL3 (or SDL2/Win32) backend
                     вҶ’  Platform_Event_Handler()      вҶҗ AHP idle, Replay idle
                     вҶ’  MoX (Input/Mouse/Fields, MOX2 main loop)
                     вҶ’  MoM (current screen: MainMenu вҶ’ MainScr вҶ’ CityScr вҶ’ вҖҰ)
                     вҶ’  back through MoX вҶ’ Platform вҶ’ SDL вҶ’ screen + audio

2. HeMoM scripted for testing (CI / regression)

HeMoM.exe --newgame ReMoM.ini --seed N  (+ optional --record game.RMR or AHP scenario file)
   вҶ’  ReMoM_Init  (no logos, no menu, no music)
   вҶ’  Skip menus, fabricate NewGame state, jump into Screen_Control()
   вҶ’  Frame loop:
        Platform_Headless.Event_Handler()
          вҶ’ AHP_Frame()         injects scripted key/click  [6]
          вҶ’ Replay (record)     appends frame to .RMR        [5]
          вҶ’ MoX вҶ’ MoM advances turns / screens
   вҶ’  On finish: Game_Save_Dump вҶ’ assertion file
                 (NONDET-tagged, since save format has uninit regions)
   вҶ’  Exit code drives ctest pass/fail.

3. ReMoM script вҖ” visual confirmation that a script does what we think

ReMoM.exe --replay game.RMR     (or load the same .hms via AHP)
   вҶ’  SDL3 backend:   real window, real audio, real timing
   вҶ’  Platform_Event_Handler():
        Replay (replay-mode) feeds the recorded frames        [5]
          вҖ” OS events are ignored; the .RMR is authoritative.
        OR AHP_Frame() injects the same scripted actions      [6]
   вҶ’  Engine and rendering proceed normally вҖ” you watch the
      AI / human-script drive the UI on screen, in real time,
      and visually verify the headless run wasn't lying.
Same scenario file, three execution modes вҖ” that's the point of putting both Replay and AHP at the Platform boundary instead of inside MoX or MoM.
